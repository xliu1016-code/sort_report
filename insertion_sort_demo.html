📚 什麼是插入排序法？

插入排序法是一種簡單直觀的排序演算法。它的工作原理如同我們整理撲克牌一樣：

  • 從第二個元素開始，將其與前面已排序的元素比較
  • 找到適當的位置並插入該元素
  • 重複此過程，直到所有元素都被排序

📝 演算法步驟

  1. 初始化： 假設陣列的第一個元素已排序
  2. 選擇元素： 從第二個元素開始逐個選擇
  3. 比較： 將選中的元素與前面已排序的元素從右到左比較
  4. 移動： 將比它大的元素向右移動一位
  5. 插入： 找到正確位置後，將元素插入
  6. 重複： 繼續選擇下一個未排序的元素，重複步驟 3-5
  7. 完成： 當所有元素都被處理後，排序完成

💻 虛擬程式碼

procedure insertionSort(arr[]) for i = 1 to length(arr) - 1 do key = arr[i] j = i - 1 while j >= 0 and arr[j] > key do arr[j + 1] = arr[j] j = j - 1 end while arr[j + 1] = key end for end procedure

📊 時間與空間複雜度

⏱️ 時間複雜度

最佳情況： O(n) - 陣列已排序

平均情況： O(n²)

最差情況： O(n²) - 反向排序

💾 空間複雜度

空間複雜度： O(1)

插入排序是原位排序（in-place），只需常數額外的空間

⚖️ 優缺點

✅ 優點

  • 實現簡單直觀
  • 對小型資料集效率較高
  • 是穩定排序
  • 原位排序（不需額外空間）
  • 適合幾乎已排序的資料

❌ 缺點

  • 對大型資料集效率較低
  • 平均時間複雜度為 O(n²)
  • 需要多次移動元素
  • 不適合非常大的資料集
💡 提示： 插入排序在資料量較小（通常少於 50 個元素）或資料基本有序時表現很好。許多高效排序演算法（如快速排序）在處理小子陣列時會使用插入排序來優化性能。

🎯 應用場景

  • 小規模排序： 資料量少於 50 個元素
  • 基本有序的資料： 對於基本排序好的資料效率高
  • 線上排序： 可以在接收資料時同時進行排序
  • 混合排序： 作為快速排序和歸併排序的子程序
  • 教學用途： 易於理解和實現的排序演算法
